# pipeline_v2 — data directory

The real data (~900 MB) is **not** in this repository. Only a small sample of the output corpus is
committed; everything else is gitignored and regenerated by running the pipeline.

## Committed

| Path | Size | What it is |
|---|---|---|
| `samples/sample_pairs.jsonl` | 120 KB | First 200 aligned MK↔SQ pairs from the full corpus, to show the output format |

## Regenerated (gitignored)

```
data/
├── extracted/            # per-PDF extracted MK/SQ text
├── segmented/            # documents split into numbered items
├── aligned/              # 257 MB — sentence-aligned pairs
├── output/               # 198 MB — final corpus (csv/tsv/jsonl, train/test splits)
├── rejected_pairs.jsonl  # 436 MB — pairs dropped by the filters, kept for analysis
└── skipped_pdfs.jsonl    # PDFs the gatekeeper rejected, with reasons
```

## How to rebuild

From `pipeline_v2/`:

```bash
pip install -r requirements.txt
python pipeline.py
```

Input PDFs are read from `../pipeline_v1/data/pdfs/` (see `pdf_dir` in `config.py`), so run the
v1 downloader first — or point `pdf_dir` at your own PDF directory.
