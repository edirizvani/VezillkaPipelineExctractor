# pipeline_v1 — data directory

The real data (~32 GB) is **not** in this repository. Only the scraper catalogs and two sample
PDFs are committed; everything else is gitignored and regenerated by running the pipeline.

## Committed

| Path | Size | What it is |
|---|---|---|
| `export/vezilka_mk_sq_full_meta.tsv` | 120 MB | v1 corpus, 100,898 pairs — **Git LFS** |
| `export/vezilka_mk_sq_cleaned.tsv` | 75 MB | v1 corpus after semantic filtering, 83,785 pairs — **Git LFS** |
| `catalog.json` | 1.6 MB | Scraped issue catalog (2001–present), used by the downloader |
| `catalog_1944_2000.json` | 3.6 MB | Scraped issue catalog for the 1944–2000 archive |
| `samples/*.pdf` | 3.3 MB | Two bilingual issues, used as fixtures by `pipeline_v2/tests/` |

The two exports are **Git LFS** objects — run `git lfs pull` to fetch them. They are superseded by
the v2 corpus; see [`DATASET.md`](../../DATASET.md). The other `export/` files (csv/json/jsonl
copies, and `vezilka_mk_sq.tsv`, which is byte-identical to `full_meta.tsv`) are gitignored as
redundant.

## Regenerated (gitignored)

```
data/
├── pdfs/       # 31 GB — downloaded PDFs, laid out as pdfs/<year>/<month>/<hash>.pdf
├── extracted/  # per-PDF extracted MK/SQ text
├── aligned/    # sentence-aligned pairs
└── export/     # 1.1 GB — final corpus exports (csv/tsv/jsonl)
```

## How to rebuild

From `pipeline_v1/`:

```bash
pip install -r requirements.txt
python pipeline.py          # scrape catalog -> download PDFs -> extract -> align -> export
```

The scraper pulls from https://slvesnik.com.mk. Downloading the full PDF set takes hours; the
committed catalogs let you skip the catalog-scraping stage.
